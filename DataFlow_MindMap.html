Backend consumes Keycloak data via:
SYNC OIDC / JWT on every HTTP request
SYNC Token introspection (optional re-validation)
READ Admin REST API (user management, role queries)
Backend NEVER writes to Keycloak's internal DB directly.

Outbound: Backend → Jira (WRITE)

DataSource ModulePurpose
New epicsD2: Spec CaptureProblem → auto-generated epics
New storiesD2: Spec CaptureEpic → auto-generated stories + AC
Labels / metadataD1E: MetadataSync domain tags back to Jira
Status transitionsD1C: GovernanceUpdate story status on spec sign-off

Backend interacts with Jira via:
SYNC Jira REST API v3 (read issues, create epics/stories)
ASYNC Jira Webhooks → Redpanda → Backend (status change events)
PM authors initiatives, epics & stories in Jira. No ADRs in Jira.

Immutability Enforcement:
Writer roles: ci_audit_writer + app_audit_writer — INSERT only (no UPDATE/DELETE grants).
Chain verification: prev_hash = SHA-256(previous row). Any edit/deletion breaks the chain.
Two emitters only: CI pipeline (GitHub Actions) + APP UX approvals. No other write path.

Keycloak → Backend

  • SYNC JWT on every request
  • READ Token introspection
  • READ User info (/userinfo)
  • READ Admin API (role queries)
  • Backend never writes to Keycloak DB

Jira ↔ Backend

  • READ Stories, epics, AC, labels
  • WRITE Create epics/stories (D2)
  • WRITE Update status on sign-off
  • ASYNC Webhooks → events
  • PM authors in Jira; no ADRs in Jira

PostgreSQL ↔ Backend

  • READ WRITE d1, d2, d3 via Prisma
  • APPEND audit schema (INSERT only)
  • No cross-schema FKs
  • Reversible migrations only
  • Dual-write: artefact + audit atomically
